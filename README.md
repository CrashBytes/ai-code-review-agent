# AI Code Review Agent

Production-ready AI code review automation using Claude API and GitHub Actions. Automatically analyzes pull requests for security vulnerabilities, code quality issues, and architectural concerns.

## Features

- Automated code review on every pull request
- Security vulnerability detection (SQL injection, XSS, auth bypasses)
- Code quality analysis (complexity, duplication, naming)
- Architectural feedback (SOLID principles, coupling, scalability)
- Performance issue identification (N+1 queries, inefficient algorithms)
- Error handling validation (try-catch, promise rejections)
- Configurable severity thresholds
- Smart finding deduplication
- Cost-effective token management
- Production monitoring and metrics

## Quick Start

### 1. Install in Your Repository

```bash
cd your-repo
npm install @anthropic-ai/sdk @octokit/rest
```

### 2. Add GitHub Actions Workflow

Create `.github/workflows/ai-code-review.yml`:

```yaml
name: AI Code Review

on:
  pull_request:
    types: [opened, synchronize, reopened]

permissions:
  contents: read
  pull-requests: write
  
jobs:
  review:
    runs-on: ubuntu-latest
    timeout-minutes: 10
    
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
          
      - uses: actions/setup-node@v4
        with:
          node-version: '18'
          cache: 'npm'
          
      - run: npm ci
        
      - name: Run AI code review
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        run: npx ai-code-review
```

### 3. Configure Secrets

1. Get your Anthropic API key from https://console.anthropic.com
2. Go to your repository Settings â†’ Secrets and variables â†’ Actions
3. Add `ANTHROPIC_API_KEY` with your API key
4. `GITHUB_TOKEN` is automatically provided

### 4. Customize Configuration (Optional)

Create `.ai-review-config.json`:

```json
{
  "model": "claude-sonnet-4-20250514",
  "minSeverity": "medium",
  "reviewFocus": [
    "security-vulnerabilities",
    "code-quality",
    "architectural-concerns",
    "performance-issues",
    "error-handling"
  ],
  "excludePatterns": [
    "**/node_modules/**",
    "**/dist/**",
    "**/build/**"
  ]
}
```

## Configuration Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `model` | string | `claude-sonnet-4-20250514` | Claude model to use |
| `maxTokens` | number | `4000` | Maximum tokens per response |
| `temperature` | number | `0.0` | Model temperature (0 = deterministic) |
| `minSeverity` | string | `medium` | Minimum severity: `low`, `medium`, `high`, `critical` |
| `reviewFocus` | array | All areas | Review focus areas |
| `excludePatterns` | array | Common artifacts | File patterns to exclude |

## Review Focus Areas

- `security-vulnerabilities`: SQL injection, XSS, authentication bypasses
- `code-quality`: Duplication, complexity, naming, magic numbers
- `architectural-concerns`: SOLID violations, coupling, abstractions
- `performance-issues`: N+1 queries, inefficient algorithms
- `error-handling`: Missing try-catch, unhandled promises

## Troubleshooting

### No comments appear

1. Check GitHub Actions logs for errors
2. Verify `ANTHROPIC_API_KEY` is configured
3. Ensure workflow has `pull-requests: write` permission
4. Check file exclusion patterns aren't too broad

### 422 errors in logs

GitHub API returns 422 when comment line numbers don't exist in the diff. This is normal for files with extensive changes. The agent logs warnings but continues.

### High costs

1. Reduce `maxTokens` in configuration
2. Add more exclusion patterns
3. Increase `minSeverity` threshold
4. Review file chunking strategy
